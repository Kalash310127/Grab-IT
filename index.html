Dual Role System

Built for Users & Admins

GRAB-IT has two completely separate portals — a customer-facing shopping experience and a powerful admin dashboard, both running from a single Java console application.

🛍️
User Portal
Customers register via mobile OTP, browse products, manage their cart, and checkout with UPI or Cash on Delivery — with auto-generated order bills saved as .txt files.
  • Mobile number + OTP registration
  • Search products by name
  • Browse by category
  • Add to cart & view cart
  • UPI or COD payment
  • Auto-generated order bill (.txt)
  • Order & payment history
  • Address book management
  • Order tracking
⚙️
Admin Dashboard
Admins get full control over products, categories, customers, and delivery partners — plus store analytics like top-selling products, low stock alerts, and sales reports.
  • Product CRUD (with Linked List)
  • Category management (BST)
  • Customer list viewer
  • Delivery partner management
  • Today's orders viewer
  • Total sales dashboard
  • Top selling products (HashMap)
  • Low stock alerts (stock ≤ 10)
  • Full sales report generator
Key Features

What makes GRAB-IT special

Every feature was engineered with real-world e-commerce logic and backed by data structures for performance.

📱
OTP Mobile Verification
New users are verified via a randomly generated 4-digit OTP. Phone must be 10 digits starting with 9. Existing users are recognized automatically by mobile lookup.
🛒
Cart with Stock Validation
Cart items are persisted in MySQL and validated against live stock levels. Users can't add more items than what's available in the database.
💳
Polymorphic Payment System
Abstract PaymentHandler class with UpiPaymentHandler and CodPaymentHandler subclasses — demonstrating polymorphism and strategy design pattern.
🧾
Auto Bill Generation
On successful payment, a formatted .txt bill is auto-generated with order ID, customer details, itemized products, total, and payment method.
📊
Sales Analytics
Admin can view today's orders, total sales with a formatted table, top-selling products ranked by quantity using Java Streams, and low-stock alerts.
🚚
Delivery Partner Management
Admins can add, view, and remove delivery partners with name and phone — stored in a dedicated MySQL table linked to the order system.
📍
Address Book
Users can store multiple delivery addresses (Home/Work) with full address and phone, and select from their address book at checkout.
🔍
Product Search
LIKE-based MySQL query lets users search for products by name with partial matches — displays ID, price, stock, and category together.
📧
Email Validation
Regex-based Gmail validation ensures only valid @gmail.com addresses are registered — rejects all other formats with an error message.
Data Structures Used

DSA-backed Architecture

The project uses three custom data structures alongside MySQL for in-memory operations — demonstrating real-world DSA application.

Linked List
SinglyLinkedList → ProductsWhen admin adds a product, it's added to both MySQL AND a custom Singly Linked List for in-session tracking. Supports addLast and display operations.
BST
CategoryBST → CategoriesAll categories are loaded from MySQL into a custom Binary Search Tree. Supports insert, search by ID, delete (with successor), and in-order traversal display.
HashMap
HashMap → Top ProductsJava's HashMap aggregates product sales quantities from the orders table, then Java Streams sort entries by value descending to rank top sellers.
Polymorphism
Abstract PaymentHandlerAbstract class with processOrder() template method. UpiPaymentHandler and CodPaymentHandler override executePayment() — OOP strategy pattern in action.
